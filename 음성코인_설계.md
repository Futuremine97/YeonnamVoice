# VoiceCoin — 음성 저작권 토큰 설계 (토크노믹스 · 아키텍처 · 규제)

> **버전** 1.0 · **작성일** 2026-06-13
> ⚠️ 본 문서는 기술 설계이며 투자 권유·법률 자문이 아닙니다. 실제 발행 전 변호사·보안감사·세무 검토가 필수입니다.

---

## 1. 개요

영어 회화 학습 서비스 "English For Us"의 **음성 저작권**을 온체인으로 표현한다. 핵심 아이디어:

- 원어민의 목소리는 **민감한 생체정보이자 저작·인격적 권리**의 대상이다.
- 그 목소리의 **소유권과 동의**를 NFT로 위변조 없이 기록하고,
- 사용(라이선스·대화 세션)에 따른 **로열티를 토큰으로 자동 분배**하며,
- 토큰 보유자가 **거버넌스**에 참여한다.

## 2. 토큰 구성

| 자산 | 표준 | 역할 |
|------|------|------|
| **VOICE** | ERC-20 | 결제·로열티·스테이킹의 단일 통화 |
| **VoiceID** | ERC-721 | 목소리 1개 = NFT 1개. 소유권 + 동의 프로비넌스 |

### VOICE 토큰 파라미터(예시)
- 최대 발행량(cap): 10억
- 초기 발행: 1억 (트레저리)
- 추가 발행: 민터 권한(예: 사용 보상)으로 상한 내에서만

### 분배(예시·조정 가능)
생태계 보상 40% · 트레저리/운영 20% · 팀(베스팅) 15% · 유동성 15% · 초기 기여자 10%. 팀·기여자 물량은 베스팅(클리프+선형)으로 잠금.

## 3. 4대 용도

1. **라이선스 결제** — 학습자가 `buyLicense(voiceId)`로 VOICE를 지불하고 목소리 사용 권리를 얻는다.
2. **사용량 기반 로열티** — 대화 세션마다 `paySession(voiceId)`로 소액을 지불, 소유자 로열티로 분배.
3. **소유권·동의 프로비넌스** — `VoiceRegistry` NFT가 `consentHash`(off-chain 동의서의 sha256), 최초 등록자, 등록 시각, 억양을 영구 기록. **동의 해시가 없으면 등록 자체가 불가**하여 "동의 없는 목소리의 토큰화"를 컨트랙트가 차단.
4. **스테이킹·거버넌스** — VOICE를 스테이킹하면 제안·투표에 스테이크 비례 가중치로 참여(시그널링).

## 4. 결제·로열티 흐름

```
학습자 ──approve VOICE──► VoiceLicense
  buyLicense(voiceId):  fee 분배 → 소유자 royaltyBps% (pull) + 트레저리 나머지(push)
  paySession(voiceId):  usagePrice 분배 → 동일 비율
소유자 ──withdrawRoyalties()──► 누적 로열티 인출
```

- 소유자 몫은 **pull 패턴**(`royaltiesOwed`)으로 재진입·실패 위험을 줄임.
- 플랫폼 수수료는 트레저리로 즉시 이체.
- 로열티 비율 `royaltyBps`는 목소리별로 소유자가 설정(상한 100%).

## 5. 온체인 ↔ 오프체인 하이브리드 아키텍처

전 사용자가 매 대화마다 가스비를 내는 건 비현실적이다. 권장 구조:

- **온체인**: 소유권·동의 프로비넌스(NFT), 라이선스 보유 사실, 정산(로열티 인출), 거버넌스.
- **오프체인(앱 서버)**: 실시간 대화·STT/TTS·사용량 집계. 동의서의 해시만 온체인에 기록.
- **정산 주기화**: 세션 사용료는 오프체인에서 누적 → 주기적으로 묶어 온체인 반영(또는 사용자가 충전한 크레딧에서 차감 후 정산).
- **가스 최적화**: L2(예: Base, Arbitrum) 또는 메타트랜잭션(가스 대납)으로 사용자 부담 제거.

앱의 기존 마켓플레이스 매핑:

| 앱(off-chain) | 온체인 |
|----------------|--------|
| 음성 등록 + 5개 동의 | `registerVoice(consentHash, …)` |
| 목소리 구매(Entitlement) | `buyLicense(voiceId)` |
| 대화 1회 사용 | `paySession(voiceId)` |
| 제공자 정산(Payout) | `withdrawRoyalties()` |

## 5-1. 목소리 분할 토큰화 (VoiceShares)

목소리 하나를 **여러 명이 분할 소유**할 수 있게 토큰화한다.

- **구조**: 소유권 NFT(VoiceRegistry tokenId)를 `VoiceShares` 금고에 잠그고, 그 목소리에 대응하는 **ERC-1155 지분 토큰**을 `shares`개 발행한다. (voiceId = ERC-1155 token id)
- **분할 소유·거래**: 지분 토큰은 양도 가능 → 투자자·팬·팀원이 한 목소리의 지분을 나눠 보유.
- **로열티 배당**: 라이선스·세션 로열티(VOICE)를 `depositRoyalty(voiceId, amount)`로 예치하면 **지분 비례로 배당**된다. 누적 분배 모델(accRewardPerShare)이라 지분이 거래돼도 정확히 정산되고, 보유자는 `claim`으로 인출.
- **통합(redeem)**: 한 주소가 지분 100%를 모으면 `redeem`으로 지분을 소각하고 NFT를 회수.
- **License 연동**: 분할된 목소리는 NFT 소유자가 `VoiceShares` 금고이므로, 플랫폼이 해당 목소리 로열티를 `depositRoyalty`로 라우팅하면 지분 보유자들에게 분배된다.

> 규제 주의: 분할 지분이 "수익 배분형"이면 **증권성**(투자계약) 위험이 커진다(특히 한국). 분할 토큰 발행·판매 전 법률 검토가 더욱 중요하다.

## 6. 거버넌스

- 스테이킹 시 의결권 = 스테이크 수량.
- 제안 최소 스테이크 요건(스팸 방지).
- 투표는 1주소 1회, 기간 종료 후 `closeProposal`로 결과 확정(시그널링).
- 발전 방향: OpenZeppelin Governor + Timelock으로 **실행 가능한** 온체인 거버넌스로 확장.

## 7. 리스크 & 규제 (반드시 검토)

1. **증권성**: 토큰이 "투자 수익 기대"로 판매되면 증권으로 분류될 수 있음(Howey 등). 유틸리티 중심 설계·표현, 수익 약속 금지.
2. **한국 가상자산 규제**: 가상자산이용자보호법 등 — 거래소 상장·발행·마케팅 규제, 실명확인·트래블룰. 국내 발행/판매는 특히 신중.
3. **AML/KYC**: 라이선스 결제·환금성 있는 토큰은 자금세탁방지 의무 유발 가능.
4. **개인정보·생체정보**: 음성은 민감정보. 음성 데이터 자체는 **절대 온체인에 올리지 않음**(불가역·공개). 해시·메타데이터만. 삭제권(개인정보보호법/GDPR)과 블록체인 불변성의 충돌 → 원본·지문은 오프체인 저장 후 삭제 가능하게, 온체인은 해시만.
5. **딥페이크·사칭**: 동의 프로비넌스로 완화하되, 권리자 신고·테이크다운·NFT 비활성화(`updateConfig active=false`) 절차 병행.
6. **스마트컨트랙트 보안**: 재진입·정수 오버플로우·권한 관리. 외부 감사 필수. 본 코드는 미감사.
7. **세금**: 토큰 수취·로열티는 과세 대상일 수 있음.
