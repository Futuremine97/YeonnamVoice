# 웹사이트 배포 가이드 (Yeonnam Voice / English For Us)

하나의 Python 웹서비스로 **API + 화면(`/app`)** 을 함께 배포합니다. 서버는 `PORT` 환경변수를 읽고 `0.0.0.0`에 바인딩하므로 대부분의 호스팅에서 그대로 동작합니다. 호스팅이 주는 **HTTPS** 덕분에 마이크·음성 기능도 정상 작동합니다(브라우저는 https에서만 마이크 허용).

> 전제: 먼저 코드를 GitHub(`Futuremine97/YeonnamVoice`)에 push 해두세요. 아래 호스팅은 그 저장소를 연결해 자동 배포합니다.

배포 후 접속 주소: `https://<your-app>.onrender.com/app`

---

## ⭐ Vercel(프런트) + Render(백엔드) — 분리 배포

Vercel은 **상시 실행 서버를 못 돌립니다**(서버리스라 메모리 상태가 매 요청 초기화). 그래서 화면만 Vercel, 백엔드는 Render에 둡니다.

### 1단계 — 백엔드를 Render에 배포
아래 "방법 A — Render.com" 그대로 진행 → `https://yeonnam-voice.onrender.com` 같은 주소 확보.

### 2단계 — 프런트가 그 백엔드를 보게 설정
`frontend/config.js` 한 줄을 Render 주소로 수정(끝에 `/` 없이):
```js
window.EFU_API = "https://yeonnam-voice.onrender.com";
```
커밋·푸시.

### 3단계 — 프런트를 Vercel에 배포
1. https://vercel.com 가입 → **Add New → Project** → `YeonnamVoice` 저장소 선택.
2. 설정:
   - **Root Directory: `frontend`** (중요)
   - Framework Preset: **Other**
   - Build Command: 비움 / Output Directory: 비움 (정적 파일 그대로)
3. Deploy → `https://<프로젝트>.vercel.app` 접속.

> 동작 원리: Vercel의 정적 화면이 `config.js`의 주소(Render 백엔드)로 API를 호출합니다. 둘 다 HTTPS라 마이크·음성도 정상. 백엔드 CORS는 이미 모두 허용(`*`)이라 교차 출처 호출이 됩니다(운영 시 Vercel 도메인으로 제한 권장).

### 포함 파일
- `frontend/config.js` — 백엔드 주소 한 줄 설정(로컬은 빈 값으로 두면 자동).
- `frontend/vercel.json` — 정적 서빙 + config.js 무캐시.

## 방법 A — Render.com (무료·추천)

1. https://render.com 가입 후 GitHub 연결.
2. **New → Blueprint** 선택 → `YeonnamVoice` 저장소 선택 → 저장소의 `render.yaml`을 자동 인식 → Apply.
   - (또는 **New → Web Service** 수동 설정)
     - Runtime: **Python**
     - Build Command: `pip install -r requirements.txt`
     - Start Command: `python backend/server.py`
     - Plan: Free
3. 배포가 끝나면 `https://yeonnam-voice.onrender.com/app` 로 접속.

### AI 답변 엔진 켜기(선택)
Render 대시보드 → 서비스 → **Environment** 에서 추가:
- `EFU_USE_AI = 1`
- `EFU_AI_PROVIDER = anthropic` (또는 `openai`, `openai_compat`)
- `EFU_AI_API_KEY = <키>` (비밀로 저장)
- (로컬/호환) `EFU_AI_BASE_URL = https://...`

> 키는 코드/저장소에 넣지 말고 **대시보드 환경변수로만** 설정하세요.

## 방법 B — Railway

1. https://railway.app → New Project → Deploy from GitHub repo → `YeonnamVoice`.
2. Start Command: `python backend/server.py` (Procfile 자동 인식되기도 함).
3. 생성된 도메인 + `/app` 로 접속. 환경변수는 Variables 탭에서 설정.

## 방법 C — Fly.io / VPS (직접 운영)

- 어디서든 다음만 되면 동작: Python 3.12, `PORT` 제공, `python backend/server.py` 실행, 80/443 → 앱 포트 프록시(HTTPS 필수).
- VPS 예: Nginx(HTTPS, Let's Encrypt) → `127.0.0.1:8000` 리버스 프록시, `python backend/server.py`를 systemd 서비스로 상시 실행.

## 배포 전 꼭 알아둘 점

1. **데이터 영속성 없음**: 현재 서버는 메모리 DB라 재시작/슬립 시 사용자·구매·등록이 초기화됩니다. 데모/MVP엔 충분하지만, 실서비스는 DB(예: Postgres)와 영속 스토리지가 필요합니다(`backend/app`의 FastAPI+SQLite 버전을 확장 권장).
2. **HTTPS 필수**: 마이크(getUserMedia)·음성인식은 https에서만 동작합니다. Render/Railway는 자동 https라 OK. 직접 VPS면 인증서를 꼭 붙이세요.
3. **CORS**: 현재 `*` 허용(프로토타입). 운영 시 도메인 화이트리스트로 제한하세요(`server.py`의 CORS 헤더).
4. **비용/한도**: Render 무료 플랜은 일정 시간 미사용 시 슬립 → 첫 접속이 느릴 수 있습니다.
5. **음성복제·코인**: 실제 음성 복제(Chatterbox)·온체인 발행은 별도 인프라이며, 가상자산·개인정보(생체) 규제 검토가 선행돼야 합니다.
6. **시크릿 관리**: API 키·지갑 키는 저장소에 절대 커밋하지 말고 호스팅 환경변수로만 주입(이미 `.gitignore`로 `.env` 제외).
