# 음성 마켓플레이스 · 구매 설계 (English For Us)

> **버전** 1.0 · **작성일** 2026-06-13
> 원어민 영어 목소리를 **프로필 단위로 등록**하고, 학습자가 **선택·구매**해 대화에 사용하는 모델.

---

## 1. 개념 모델

```
User ──owns──► Voice ──listed──► (Catalog 노출)
User ──purchase──► Order ──paid──► Entitlement(보유권) ──grants──► Voice 사용권
```

- **Voice(목소리 프로필)**: 원어민 화자 1명의 복제 목소리. 메타데이터(이름·억양·성별·설명·가격·샘플)를 가진다.
- **Catalog(스토어)**: `is_listed=true` 이고 `status=ACTIVE`(동의 완료)인 목소리만 노출.
- **Order(주문)**: 구매 시작 시 생성. 결제 provider 인텐트와 연결.
- **Entitlement(보유권)**: 결제 완료 시 부여. 이 권리가 있어야 대화에서 그 목소리를 쓸 수 있다.

> **동의와의 관계(중요)**: 판매 등록(publish)은 **동의 완료된 ACTIVE 음성에만** 허용된다. 동의 게이트를 통과하지 못한 목소리는 절대 상품화되지 않는다. 즉 "타인 목소리를 동의 없이 팔 수 없다"가 코드로 강제된다.

---

## 2. 구매 플로우

```
① 스토어 조회        GET  /catalog/voices?user_id=...
② 구매 시작          POST /catalog/voices/{id}/purchase   → Order(PENDING) + payment_token
③ 결제 확정          POST /catalog/orders/{id}/confirm?payment_token=...
                     → 결제 성공 시 Order(PAID) + Entitlement 생성
④ 사용               POST /chat (voice_id)   → 보유권 검증 통과 시 합성
```

- 이미 보유(소유 또는 entitlement)한 목소리는 재구매가 409로 차단된다.
- **무료(0원)** 목소리도 동일 플로우(주문→확정)로 보유권을 부여해 사용 이력을 일관되게 관리한다.

### 결제 추상화

`services/payments.py`의 `PaymentProvider` 인터페이스 뒤로 결제를 숨긴다. 프로토타입은 `MockPayments`(즉시 성공). 실서비스는 **Stripe**(`create_intent` → client_secret, 웹훅으로 `confirm`) 등으로 교체. 웹훅 기반 확정으로 바꾸면 클라이언트 토큰 노출 없이 서버가 결제 완료를 신뢰할 수 있다.

## 3. API 요약

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/voices` | 음성 업로드 + 동의 (ACTIVE 후보) |
| POST | `/catalog/voices/{id}/publish` | ACTIVE 목소리를 스토어에 판매 등록 |
| GET | `/catalog/voices` | 판매 목록 (user_id 주면 보유 여부 표시) |
| POST | `/catalog/voices/{id}/purchase` | 주문 생성 + 결제 인텐트 |
| POST | `/catalog/orders/{id}/confirm` | 결제 확정 → 보유권 부여 |
| GET | `/catalog/my-voices` | 사용 가능한 목소리(소유+구매) |
| POST | `/chat` | 대화 (보유권 없으면 402) |

## 4. 판매자(목소리 제공자) 측 고려

원어민이 자기 목소리를 등록·판매하는 양면 마켓을 전제로 한다.

- **수익 배분(정산)**: 판매가에서 플랫폼 수수료를 제외하고 제공자에게 정산. (모델 확장 시 `Payout` 엔티티 추가.)
- **라이선스 범위**: 구매자는 "학습 대화 합성"에 한해 사용 권리를 얻는다. 재배포·재판매·다운로드는 불가(스트리밍 합성만). 약관에 명시.
- **제공자 동의 철회**: 제공자가 목소리를 내리면(`is_listed=false`) 신규 판매는 중단되되, 기존 구매자 권리·환불 정책을 약관으로 규정.
- **품질 검수**: 등록 시 미리듣기 샘플·억양·성별 메타 검증, 부적절 음성 반려.

---

## 5. 악용·부정거래 방지 (마켓 특화)

- 동의 미완료 목소리 판매 불가(코드 강제).
- 사칭 의심(유명인 유사) 목소리는 판매 전 검토 큐로.
- 구매 후에도 **대화 입력 모더레이션**과 **합성 워터마크·레이트리밋**은 동일하게 적용된다.
- 결제 사기·칩백 대응은 결제 provider(Stripe Radar 등)에 위임.

## 6. 데이터 모델 (구현됨)

`backend/app/models.py`

- `Voice`: `is_listed, price_cents, accent, gender, description, preview_path` 추가.
- `Order`: `user_id, voice_id, amount_cents, status(PENDING/PAID/FAILED/REFUNDED), provider, provider_ref`.
- `Entitlement`: `user_id, voice_id, order_id, active`.

대화 시 `chat._user_can_use()`가 **소유 또는 보유권**을 확인하고, 없으면 402를 반환한다.
