# 동의 · 악용방지 기능 설계 (English For Us)

> **버전** 1.0 · **작성일** 2026-06-13
> 음성복제 서비스의 최대 리스크는 기술이 아니라 **타인 목소리의 무단 복제·사칭**입니다. 이 문서는 그 리스크를 제품 기능으로 통제하는 방법을 정의하고, 프로토타입 백엔드에 실제로 구현된 로직을 명세합니다.

---

## 1. 설계 원칙

1. **동의 없는 음성은 지문화하지 않는다.** 업로드만으로는 voice_id가 발급되지 않고, 동의 절차를 통과해야만 활성화된다.
2. **모든 합성 음성은 추적 가능해야 한다.** 생성 오디오에 식별자(메타데이터 + 가청 한계 워터마크/태그)를 남긴다.
3. **삭제는 사용자의 권리다.** 음성 원본·지문·생성물은 언제든 완전 삭제 가능.
4. **악용 입력은 합성 전에 차단한다.** 사칭·혐오·괴롭힘·미성년 대상화 등은 텍스트 단계에서 거른다.
5. **거부는 친절하게, 기록은 확실하게.** 차단 사유를 사용자에게 안내하고, 감사 로그(audit log)를 남긴다.

---

## 2. 동의(Consent) 기능

### 2.1 동의 게이트 — 음성 등록 상태 머신

음성은 다음 상태를 거친다. `ACTIVE`가 되어야만 대화에 사용 가능.

```
PENDING_CONSENT  ──(동의 통과)──►  VERIFYING  ──(검증 통과)──►  ACTIVE
       │                              │
       └─(거부/미동의)─► REJECTED      └─(검증 실패)─► REJECTED
ACTIVE ──(사용자 삭제)──► DELETED
```

### 2.2 동의 수집 항목 (업로드 시 필수)

- **본인/권한 확인**: "이 음성은 (a) 나 자신의 것이거나 (b) 녹음 대상 본인에게 명시적 동의를 받았다."
- **목적 제한 동의**: 영어 학습 대화 합성 용도로만 사용.
- **사칭 금지 서약**: 유명인·정치인·타인 사칭, 사기, 허위정보 생성에 사용하지 않는다.
- **미성년자 아님 확인**: 음성 주인이 만 18세 이상(또는 보호자 동의).
- **약관·개인정보 처리방침 동의** (생체정보 취급 고지 포함).

각 항목은 개별 체크박스로 받고, 동의 시각·버전·해시를 저장한다(추후 증빙).

### 2.3 검증(VERIFYING) 단계 — 선택적 강화

신뢰도를 높이기 위한 옵션(MVP에선 1번만, 이후 단계적 도입):

1. **자기낭독 검증(challenge phrase)**: 시스템이 임의 문장을 제시하고, 업로드 음성 또는 추가 녹음에서 해당 문구를 읽게 해 "업로더 = 화자"를 약하게 입증.
2. **유명인 음성 차단**: 알려진 공인 음성 임베딩과 유사도 비교(있을 경우)해 사칭 의심 시 보류.
3. **수동 검토 큐**: 의심 케이스는 운영자 검토로 라우팅.

## 3. 악용방지(Abuse Prevention) 기능

### 3.1 입력 모더레이션 (대화 텍스트/STT 결과)

합성·응답 **이전에** 다음을 검사한다.

| 카테고리 | 처리 |
|----------|------|
| 사칭 유도("○○인 척 말해줘") | 차단 + 안내 |
| 혐오·괴롭힘·위협 | 차단 |
| 성적·미성년 관련 부적절 | 차단 |
| 사기·피싱 대본 생성 | 차단 |
| 자해 관련 | 차단 + 도움 리소스 안내 |

구현: 키워드/패턴 1차 필터 → (고도화) LLM·전용 모더레이션 API 2차 판정. 프로토타입에는 **규칙 기반 모더레이터**를 실제 구현.

### 3.2 합성 출력 보호

- **워터마크/식별 태그**: 생성된 오디오 파일 메타데이터에 `synthetic=true`, `service`, `voice_id`, `generated_at`를 기록. (고도화: 가청 한계 워터마크 신호 삽입.)
- **레이트 리밋**: voice_id·사용자별 합성 횟수 제한으로 대량 악용 방지.

### 3.3 감사 로그(Audit Log)

동의 기록, 차단 이벤트, 삭제 요청, 합성 호출을 시각·사유와 함께 저장. 분쟁·신고 대응 근거.

### 3.4 신고 & 테이크다운

- 합성 음성/대화에 신고 버튼. 신고 시 해당 voice_id 일시 정지 → 검토.
- 권리자 요청 시 즉시 비활성화 경로.

---

## 4. 데이터 보호 & 삭제권

- 음성 원본·지문·생성물 **암호화 저장**, 접근 최소화.
- **보관기간** 설정(예: 미사용 voice 12개월 후 자동 만료 후보).
- **완전 삭제 API**: 원본·지문·캐시·생성물까지 제거하고 감사 로그에 삭제 사실 기록.
- 음성은 **민감 생체정보**로 취급(고지·동의·국가별 규제 확인 필요: 한국 개인정보보호법, 일본 APPI 등).

---

## 5. 프로토타입에 실제 구현된 항목

아래는 `backend/`에 코드로 동작하는 부분이다.

| 기능 | 구현 위치 | 동작 |
|------|-----------|------|
| 동의 게이트(상태머신) | `services/consent.py`, `models.py` | 동의 미통과 시 voice_id 비활성, 대화 거부 |
| 동의 항목 검증 | `routers/voices.py` | 5개 필수 동의 누락 시 422/거부 |
| 동의 증빙 해시·버전·시각 | `models.ConsentRecord` | 저장 |
| 입력 모더레이션 | `services/moderation.py` | 사칭·혐오·미성년·사기 패턴 차단 |
| 합성 워터마크 메타데이터 | `services/tts.py` | 생성물에 synthetic 태그 부착 |
| 레이트 리밋 | `services/moderation.py` | voice별 합성 횟수 제한 |
| 감사 로그 | `models.AuditLog` | 동의·차단·삭제·합성 이벤트 기록 |
| 완전 삭제 | `routers/voices.py` DELETE | 지문·원본·생성물 제거 + 로그 |
| 자해 안내 | `services/moderation.py` | 차단 대신 지원 안내 메시지 |

> TTS/STT/LLM은 프로토타입에서 **mock 어댑터**로 동작하며, API 키/모델만 꽂으면 실제 엔진으로 교체되도록 추상화되어 있다.
