# 보안 체크 — 음성 등록 + 코인 발행/블록체인 연동

> 2026-06-13 · 프로토타입 자체 점검(self-review). 실서비스 전 외부 보안 감사 권장.

## 1. 키·서명 (가장 중요)

- ✅ **앱은 개인키를 절대 보관·전송하지 않는다.** 온체인 발행 서명은 전적으로 사용자 지갑(MetaMask, `window.ethereum`)에서 이뤄진다.
- ✅ `/settings`는 **공개값만** 노출(체인 ID·컨트랙트 주소·explorer). AI API 키 등 비밀값은 응답에 포함하지 않음(검증: `api_key` 필드 미노출).
- ✅ `.env`, `node_modules`, `data/`는 `.gitignore`로 커밋 제외(키·빌드 산출물 유출 방지).

## 2. 입력 검증 (서버)

| 항목 | 규칙 | 결과 |
|------|------|------|
| 컨트랙트 주소 | `^0x[0-9a-fA-F]{40}$` | 위반 시 422 |
| tx 해시 | `^0x[0-9a-fA-F]{64}$` | 위반 시 422 |
| token_id | 정수 파싱 | 위반 시 422 |
| explorer_base | `http(s)://`만 허용 | 위반 시 422 |
| 동의 항목 | 5개 모두 true | 미충족 시 403 |

→ 검증 자동 테스트 통과(10/10).

## 3. 동의·프로비넌스 무결성

- ✅ `consent_hash`는 **서버에서** 동의 기록의 정규화(sha256)로 계산 → 클라이언트가 위조 불가.
- ✅ 온체인에는 **해시만** 기록. 음성 원본·지문은 절대 체인에 올리지 않음(불변성과 삭제권 충돌 회피, 생체정보 보호).
- ✅ 동의 미통과 목소리는 등록/발행 불가(컨트랙트도 `consentHash != 0` 강제).

## 4. 웹 프런트엔드

- ✅ React가 텍스트를 자동 이스케이프(XSS 방지). `dangerouslySetInnerHTML` 미사용.
- ✅ 외부 링크는 `target="_blank" rel="noreferrer"`(tabnabbing 방지).
- ✅ explorer 링크의 base는 서버에서 http(s)로 강제된 값만 사용.
- ✅ 주소 형식은 클라이언트(`isAddr`)에서도 1차 검증 후 컨트랙트 인스턴스화.

## 5. 스마트컨트랙트 (blockchain/)

- ✅ `ReentrancyGuard`로 결제·인출·스테이킹 재진입 방지.
- ✅ 로열티는 **pull 패턴**(`withdrawRoyalties`)으로 분배 — push 실패/재진입 위험 감소.
- ✅ 권한: 민팅은 `onlyMinter`, 설정 변경은 `onlyOwner`/`onlyVoiceOwner`.
- ✅ Solidity ^0.8 (정수 오버플로우 기본 보호).
- ⚠️ **미감사**. 메인넷 전 외부 감사·테스트 커버리지 확대 필수.

## 6. 운영 전 보완 필요(프로토타입 한계)

- CORS가 `*` (로컬 개발용) → 운영은 도메인 화이트리스트.
- HTTP(cleartext) 사용 → 운영은 HTTPS + HSTS.
- 인증/세션 없음(로컬 전용) → 운영은 인증·권한·레이트리밋 강화.
- 결제(모바일)는 스토어 정책상 IAP 필요.
- 가상자산 규제(한국 가상자산이용자보호법)·증권성·AML/KYC·세무 검토 필수.
